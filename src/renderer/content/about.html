JavaScript Interview Comparison Tables

1. var vs let vs const

Feature var let const
Scope Function Block Block
Hoisting Yes (initialized as undefined) Yes (not initialized) Yes (not initialized)
Re-declaration Allowed Not allowed Not allowed
Re-assignment Allowed Allowed Not allowed
Temporal Dead Zone No Yes Yes
Global object property Yes No No

Key Points:

  • Always prefer const by default, use let when you need to reassign, and avoid var
  • const doesn't make objects immutable - just prevents reassignment of the variable
  • Temporal Dead Zone (TDZ) is the period between entering scope and being declared

2. == (Loose Equality) vs === (Strict Equality)

Comparison == ===
Type coercion Performs type conversion No type conversion
Example: 5 == '5' true false
Example: null == undefined true false
Example: 0 == false true false
Performance Slightly faster (but negligible) Slightly slower
Recommended use Rarely Always preferred

Key Points:

  • === is safer as it doesn't do type coercion
  • The only exception is when checking for null or undefined together: value == null
  • Type coercion rules can be surprising: [] == ![] is true!

3. Traditional Function vs Arrow Function

Feature Traditional Function Arrow Function
this binding Own this context Inherits from parent scope
arguments object Available Not available
Constructor Can be used as constructor Cannot be used as constructor
Implicit return No Yes (when no curly braces)
Syntax function() {} () => {}
Hoisting Fully hoisted Not hoisted

Traditional Function Example:

function greet(name) {
    console.log(`Hello, ${name}!`);
    console.log(arguments); // Arguments object
}

greet('Alice'); // Hello, Alice!
new greet('Bob'); // Works (constructor)

Arrow Function Example:

const greet = (name) => {
    console.log(`Hello, ${name}!`);
    // No arguments object
    // Can't use new
};

const square = x => x * x; // Implicit return

greet('Alice'); // Hello, Alice!
// new greet('Bob'); // TypeError

4. Array Methods: forEach vs map vs filter vs reduce

Method Purpose Returns Mutates Original Chainable
forEach Iteration undefined No No
map Transformation New array No Yes
filter Selection New array (subset) No Yes
reduce Aggregation Single value No Yes

Example Usage:

const numbers = [1, 2, 3, 4];

// forEach - just iterate
numbers.forEach(num => console.log(num));

// map - transform
const squares = numbers.map(num => num * num);

// filter - select
const evens = numbers.filter(num => num % 2 === 0);

// reduce - aggregate
const sum = numbers.reduce((acc, num) => acc + num, 0);

// Chaining example
const result = numbers
    .filter(n => n > 2)
    .map(n => n * 2)
    .reduce((a, b) => a + b);

When to Use Each:

  • forEach: When you need to perform side effects (logging, DOM manipulation)
  • map: When you need to transform each element in an array
  • filter: When you need to select elements based on a condition
  • reduce: When you need to compute a single value from the array (sum, average, etc.)
  • Chaining: Combine methods for complex transformations (map after filter, etc.)
